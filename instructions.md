# Instructions

## Ubicaciones para Archivos de Instrucciones / Locations for Instruction Files

Este repositorio `.github` es un repositorio especial de GitHub que permite configuraciones a nivel de organización.

This `.github` repository is a special GitHub repository that allows organization-level configurations.

### Ubicaciones Comunes / Common Locations

#### 1. **Raíz del Repositorio / Repository Root** (`/instructions.md`)
- **Propósito / Purpose**: Instrucciones generales para toda la organización
- **Visibilidad / Visibility**: Alta - Visible inmediatamente al abrir el repositorio
- **Uso recomendado / Recommended use**: Guías generales, políticas, normas de contribución

#### 2. **Carpeta Profile** (`/profile/README.md` o `/profile/instructions.md`)
- **Propósito / Purpose**: Contenido que aparece en la página de perfil de la organización
- **Visibilidad / Visibility**: Pública - Se muestra en https://github.com/{org-name}
- **Uso recomendado / Recommended use**: Presentación de la organización, misión, valores

#### 3. **Carpeta .github** (`/.github/instructions.md`)
- **Propósito / Purpose**: Instrucciones específicas de GitHub (workflows, actions, etc.)
- **Visibilidad / Visibility**: Media - Para usuarios que buscan configuraciones de GitHub
- **Uso recomendado / Recommended use**: Guías para workflows, templates, configuraciones

#### 4. **Carpeta docs** (`/docs/instructions.md`)
- **Propósito / Purpose**: Documentación detallada y técnica
- **Visibilidad / Visibility**: Media - Para documentación organizada
- **Uso recomendado / Recommended use**: Documentación técnica, guías detalladas, tutoriales

### Archivos Especiales en Repositorio .github / Special Files in .github Repository

Este repositorio puede contener:
- **ISSUE_TEMPLATE/** - Plantillas para issues
- **PULL_REQUEST_TEMPLATE.md** - Plantilla para pull requests
- **workflow-templates/** - Plantillas de workflows reutilizables
- **copilot-instructions.md** - Instrucciones para GitHub Copilot
- **CODE_OF_CONDUCT.md** - Código de conducta
- **CONTRIBUTING.md** - Guía de contribución
- **SECURITY.md** - Política de seguridad
- **SUPPORT.md** - Información de soporte

### Recomendación / Recommendation

Para un archivo `instructions.md` general, se recomienda crearlo en:
- **Nivel raíz** (`/instructions.md`) - Si son instrucciones generales para desarrolladores
- **Carpeta profile** (`/profile/README.md`) - Si es contenido público de presentación
- **Carpeta docs** (`/docs/instructions.md`) - Si es documentación técnica detallada

## Ejemplo de Contenido / Example Content

Un archivo `instructions.md` típico puede incluir:

### Para Desarrolladores / For Developers
```markdown
# Instrucciones de Desarrollo

## Configuración Inicial
- Pasos para configurar el entorno
- Herramientas necesarias
- Dependencias

## Flujo de Trabajo
- Cómo crear branches
- Cómo hacer commits
- Cómo crear pull requests

## Estándares de Código
- Guías de estilo
- Prácticas recomendadas
- Patrones a seguir
```

### Para Organización / For Organization
```markdown
# Instrucciones Organizacionales

## Misión y Visión
- Objetivos de la organización
- Valores fundamentales

## Políticas
- Políticas de contribución
- Código de conducta
- Proceso de revisión
```
